# Commit Messages

Use Conventional Commit format: `<type>(<scope>): <description>`

- Single line, max 72 chars, lowercase, imperative mood, no period
- type (required): feat, fix, docs, style, refactor, perf, test, chore, ci, build, revert
- scope (optional): affected component
- Do not add Co-Authored-By lines
- Include verbose description in commit body (lines after subject)

Examples: `feat(auth): add oauth2 support` | `fix(api): handle null response`
